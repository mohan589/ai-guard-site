AI Guard is an endpoint data-protection application that evaluates sensitive file access before data is returned to local AI tools, developer automation, and other processes.

Protection model

Policy at the moment sensitive files are opened.

Optional wrappers cannot cover direct file access. AI Guard uses a native authorization boundary for new sensitive-file opens and keeps policy decisions local.

01

Process attribution

Associate each covered request with process, executable, user, and responsible-process metadata.

02

Local policy

Evaluate sensitive paths locally and return an explicit allow, deny, or administrator-backed approval decision.

03

Durable audit

Record protected decisions locally with bounded, owner-controlled audit data before an authorized action proceeds.

Apple Endpoint Security usage

Narrow purpose. Explicit boundary.

The requested entitlement is used by com.aiguard.endpoint-security.

What it does

  • Subscribes to ES_EVENT_TYPE_AUTH_OPEN.
  • Evaluates new read opens using file-path and process metadata.
  • Responds with allowed flags or denies the open.
  • Protects configured locations such as environment files, private keys, cloud credentials, browser credential stores, and administrative policy.

What it does not do

  • Does not read or copy file contents through Endpoint Security.
  • Does not use Endpoint Security data for advertising or profiling.
  • Does not silently activate without user or administrator approval.
  • Does not claim complete network or API mediation in the current release.

Privacy by design

Security metadata stays controlled.

Authorization processing occurs on the endpoint. Enterprise audit delivery is optional, administrator-configured, redacted, and limited to policy and process metadata rather than sensitive file contents. This website uses no analytics, advertising, cookies, or client-side scripts.
